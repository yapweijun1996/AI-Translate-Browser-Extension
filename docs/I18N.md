# i18n guide (extension UI)

The extension UI is localized in 6 languages via the standard `chrome.i18n` mechanism. This is **separate** from the translation feature — UI language ≠ target translation language.

| Locale dir | Language |
|---|---|
| `_locales/en/` | English — **default & fallback** |
| `_locales/zh_CN/` | Mandarin (Simplified Chinese) |
| `_locales/ms/` | Malay |
| `_locales/ja/` | Japanese |
| `_locales/vi/` | Vietnamese |
| `_locales/ko/` | Korean |

## Adding or changing a string

1. Add the key to `_locales/en/messages.json`:
   ```json
   "modal_explain_button": {
     "message": "Explain",
     "description": "Button inside the translation modal that expands the learner explanation"
   }
   ```
   Always fill `description` — translators (human or LLM) depend on it.
2. Add the same key to **all five other locales in the same commit.** English text as a temporary placeholder is allowed *before* M5, but then the commit message must contain `i18n-placeholder` so we can find them later.
3. Use it:
   - JS (any context, including content scripts): `chrome.i18n.getMessage('modal_explain_button')`
   - `manifest.json`: `"name": "__MSG_ext_name__"`
   - HTML: no inline text — set it from JS at load, or use a tiny `data-i18n="key"` walker.

## Key naming

Flat keys, prefixed by surface:

| Prefix | Surface |
|---|---|
| `ext_` | manifest (name, description) |
| `popup_` | popup |
| `options_` | options page |
| `modal_` | in-page translation modal / trigger icon |
| `menu_` | context menu items |
| `error_` | error + upsell messages |

Placeholders: use chrome.i18n named placeholders, never string concatenation (word order differs across our 6 languages):

```json
"error_quota_reset": {
  "message": "Free quota resets in $HOURS$ hours",
  "placeholders": { "hours": { "content": "$1" } }
}
```

## Rules

- **No hardcoded user-visible strings anywhere** — enforced by the CI check (T-028): fails if a key is missing from any locale, or if a suspicious string literal appears in UI code.
- Don't reuse a key for a different meaning because the English happens to match — make a new key.
- Never change a key's meaning; add a new key and drop the old one.
- Dates/numbers: use `Intl.*` with the UI locale, not manual formatting.
- Text expansion: German-style 30% expansion doesn't apply here, but Vietnamese and Malay run longer than English — UI must tolerate ~40% wider strings (test options page + modal buttons).

## Testing a locale

Launch Chrome with a different UI language: `chrome.exe --lang=ja` (or change chrome://settings/languages order). `chrome.i18n.getMessage` follows the browser UI language; there is no runtime switch (a manual override is a v1.x nice-to-have — see SPEC §8).
